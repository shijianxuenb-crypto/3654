# 时间守护者 TimeKeeper 微信小程序版

这是一个可以导入微信开发者工具、GitHub 或扣子编程的微信小程序项目。当前版本已经改成更轻的待办清单工具，使用方式接近 Microsoft To Do。

## 主要功能

- 我的今天、重要、有提醒、已完成四个视图
- 一句话添加任务，例如：`下午3点提醒我开会`
- 手动编辑任务和提醒时间
- 标记完成、标记重要、单条删除
- 批量选择和批量删除
- 本地保存任务数据
- 使用 `wx.addPhoneCalendar` 把带提醒的任务写入手机日历

## 重要说明

微信小程序通常不能直接控制手机系统闹钟，也不能像 Android 原生 App 一样长期后台运行。这个版本采用更稳妥的方式：任务保存在小程序里；需要系统级提醒时，点击任务旁边的“日历”，把任务加入手机日历，由系统日历负责提醒。

## GitHub 根目录需要上传的文件

把下面这些文件放在 GitHub 仓库根目录即可：

- `README.md`
- `app.js`
- `app.json`
- `app.wxss`
- `index.js`
- `index.wxml`
- `index.wxss`
- `project.config.json`
- `sitemap.json`

不要上传 zip 压缩包，也不要把这些文件再套进子文件夹。

## 导入扣子编程

1. 打开扣子编程。
2. 选择“GitHub 导入”。
3. 选择这个仓库。
4. 项目类型选择小程序。
5. 如果要求项目目录，选择仓库根目录。
6. 导入后检查根目录是否能看到 `app.json`、`index.js`、`index.wxml`、`index.wxss`。

## 导入微信开发者工具

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择包含 `app.json` 的这个文件夹。
4. AppID 使用你的小程序测试号或正式小程序 AppID。
5. 点击“编译”查看效果。
6. 需要真机体验时，点击“预览”扫码。

## 发布为正式小程序

1. 在微信公众平台注册或使用已有小程序。
2. 确认 `project.config.json` 里的 `appid` 是你的真实 AppID。
3. 在微信开发者工具里上传代码。
4. 在微信公众平台提交审核。
5. 审核通过后发布。

## 数据提醒

任务数据保存在微信本地缓存中。换手机、清理微信缓存或卸载微信后，数据可能丢失。后续如果需要多设备同步，需要再接入云开发或后端数据库。
